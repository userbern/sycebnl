# Mémoire du projet

Cette mémoire est évolutive. Elle conserve les décisions durables et les conventions stabilisées.

## Décisions actives

- toujours utiliser Riverpod pour la gestion d'état;
- ne jamais écrire de SQL dans les widgets;
- appliquer le Repository Pattern pour l'accès aux données;
- garder Material 3 comme base d'interface;
- rester en Feature First;
- penser Desktop First;
- réutiliser les composants existants avant d'en créer de nouveaux;
- éviter les duplications;
- respecter les règles comptables SYCEBNL.

## Règles de mise à jour

- ajouter une nouvelle ligne quand une décision devient stable;
- préciser la date ou le contexte quand une règle dépend d'une itération précise;
- retirer ou marquer obsolète toute décision invalidée par l'évolution du projet;
- ne pas réécrire tout l'historique quand une seule décision change.
