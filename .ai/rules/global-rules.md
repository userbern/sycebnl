# Règles globales

## Règles non négociables

- pas de duplication inutile;
- pas de logique métier dans les widgets;
- pas de SQL dans l'UI;
- pas de mélange des responsabilités entre agents;
- code modulaire et lisible;
- fichiers raisonnablement courts;
- privilégier les composants réutilisables;
- documenter les décisions structurantes.

## Règles de collaboration

- chaque agent travaille uniquement dans son périmètre;
- l'orchestrateur coordonne, mais ne code pas;
- les validations doivent être explicites;
- toute décision importante va dans la mémoire du projet.
