# Workflow: revue de code

## Agents impliqués

- Orchestrateur
- Reviewer
- éventuellement Architecte ou QA selon les risques

## Ordre d'exécution

1. Lire le changement.
2. Évaluer le risque.
3. Lister les findings.
4. Proposer des corrections.
5. Clôturer avec un verdict.

## Critères de validation

- findings concrets;
- gravité explicite;
- absence de faux positifs majeurs;
- recommandations actionnables.

## Livrables

- rapport de revue;
- verdict;
- priorisation des corrections.
