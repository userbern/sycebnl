# Utilisation quotidienne

## Pour une nouvelle demande

1. Choisir la commande adaptée.
2. Laisser l'orchestrateur lire le contexte partagé.
3. Lancer les agents spécialisés nécessaires.
4. Valider les livrables.
5. Mettre à jour la mémoire si une décision devient durable.

## Pour étendre le système

- ajouter un contexte si une nouvelle zone d'information devient partagée;
- créer un agent si une spécialité revient souvent;
- ajouter un workflow si un type de demande se répète;
- enrichir la mémoire quand une décision est stable;
- conserver les règles globales intactes.
